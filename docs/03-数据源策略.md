# 03 - 数据源策略

> App 启动时按层级探测数据源；cc switch 是**可选增强**，不是必须。
>
> **三层优先级：L1 官方订阅 → L2 第三方 env vars → L3 cc switch**

## 设计动机

最初假设"装了 cc switch 才能用 App"，这是错的：

- 用户可能只用 Claude Code 官方订阅（Pro/Max），从未装过 cc switch
- 用户可能自己手动配了 `ANTHROPIC_BASE_URL` 走第三方代理，没装 cc switch
- 用户可能装了 cc switch，但当前没用它（手动改了 settings.json）

App 必须在所有三种情况下都能工作，**Touch Bar 显示的信息按当前激活层级决定**。

## 三层定义

### L1：官方订阅

**触发条件：**
- `settings.json` 的 `env` 块为空，**或**
- `ANTHROPIC_BASE_URL` 指向 `anthropic.com` / `claude.ai`

**用户提供什么：**
- Claude Code 通过 Anthropic 官方 API 工作
- 可能有 Claude.app 安装（订阅状态可推断）
- 没有第三方代理，所以没有"provider 名"概念（显示 "Claude Official"）

**App 怎么拿数据：**
- model：从 `settings.json` env 块的 `ANTHROPIC_DEFAULT_*_MODEL`，或默认值（`claude-sonnet-4-5` 等）
- provider 名：硬编码 "Claude Official"
- cost：**无精确值**（官方订阅是定额订阅，按 token 不计费）。显示订阅状态（Pro / Max）或 `—`
- session 状态：从 hooks 拿（任何层级都有）

### L2：第三方模型（env vars）

**触发条件：**
- `settings.json` 的 `env` 块含 `ANTHROPIC_BASE_URL`
- 该 URL **不**指向 `anthropic.com` / `claude.ai`

**用户提供什么：**
- 自己配的第三方代理（如智谱 BigModel 网关、自建 newapi 等）
- 在 env 里指定 base URL + 模型映射
- 可能装了 cc switch 但当前没启用（用户手动绕过）

**App 怎么拿数据：**
- model：从 env 块的 `ANTHROPIC_DEFAULT_*_MODEL` / `ANTHROPIC_MODEL`
- provider 名：从 `BASE_URL` 推断（hostname → 友好名映射表）
- cost：**无数据**（第三方代理不暴露 usage）。HUD 显示 `—`
- session 状态：从 hooks 拿

### L3：cc switch

**触发条件：**
- 装了 cc switch app
- cc switch 的 SQLite db 存在
- 当前有 active provider（`providers WHERE is_current=1`）

**用户提供什么：**
- 完整的 provider 管理（多 provider 注册、健康监测、failover）
- 用量记录（`usage_daily_rollups` 表）
- 模型定价表（`model_pricing` 表）
- 自动改写 `settings.json`（用户切 provider 时）

**App 怎么拿数据：**
- model / provider：从 SQLite `providers` 表
- cost：从 `usage_daily_rollups` 聚合
- 健康状态：从 `provider_health` 表
- proxy 状态：从 `proxy_config` 表
- session 状态：仍然从 hooks 拿（cc switch 不管 Claude Code 进程内的事）

## 优先级解析

```
                  ┌─────────────────────────────────┐
                  │  读 ~/.claude/settings.json     │
                  └────────────┬────────────────────┘
                               │
                  ┌────────────▼────────────────────┐
                  │  env.ANTHROPIC_BASE_URL 是第三方?│
                  └────────────┬────────────────────┘
                        yes   /   \   no
                             /     \
              ┌─────────────▼─┐   ┌─▼──────────────────┐
              │ 检测 cc switch│   │ 检测 cc switch     │
              │ 装了+激活?    │   │ 装了+激活?         │
              └─────┬─────────┘   └────────┬───────────┘
              yes/no \                    / yes/no
                     ┌───────────────────┐
                     │ 决策矩阵           │
                     └───────────────────┘

  决策矩阵（cc switch 优先级最高，因为它数据最完整）：
  ┌──────────────┬──────────────┬─────────────────┐
  │              │ cc switch=no │ cc switch=yes   │
  ├──────────────┼──────────────┼─────────────────┤
  │ env=官方     │ L1 official  │ L3 ccSwitch ★   │
  │ env=第三方   │ L2 envVars   │ L3 ccSwitch ★   │
  └──────────────┴──────────────┴─────────────────┘
  ★ = cc switch 装了就优先用（数据最完整）
```

**核心规则：** 装了 cc switch 且有 active provider → 优先 L3，覆盖 L1/L2。
原因：cc switch 改写 settings.json 时会把 env 块写好，但读 SQLite 比 parse env 更精确，且能拿 cost / health 数据。

## 实现：SourceResolver

```swift
final class SourceResolver {
    let claudeConfigDir: URL
    let ccSwitchLocator: () throws -> URL?

    func resolve() throws -> ActiveSource {
        let settings = try SettingsJsonReader.read(at: claudeConfigDir)
        let envIsThirdParty = settings.env.map { !isOfficial($0.anthropicBaseURL ?? "") } ?? false

        // 优先 cc switch
        if let dbURL = try ccSwitchLocator(),
           let db = try? CCSwitchDB.open(dbURL, readonly: true),
           let active = try? db.activeProvider(),
           active != nil {
            return .ccSwitch
        }

        if envIsThirdParty, let baseURL = settings.env?.anthropicBaseURL {
            return .envVars(baseURL: baseURL)
        }

        return .official
    }

    private func isOfficial(_ url: String) -> Bool {
        let lowered = url.lowercased()
        return lowered.isEmpty
            || lowered.contains("anthropic.com")
            || lowered.contains("claude.ai")
    }
}
```

## 各层级的数据可用性矩阵

| 数据 | L1 官方 | L2 env vars | L3 cc switch |
|---|---|---|---|
| provider 名 | 硬编码 "Claude Official" | URL → 友好名映射 | SQLite `providers.name` |
| model | env 块 or 默认值 | env 块 | env 块（cc switch 写入） |
| cost today | `—`（订阅制） | `—`（无数据） | SQLite `usage_daily_rollups` |
| cost month | `—` | `—` | SQLite 聚合 |
| 今日请求数 | hook 自算 | hook 自算 | SQLite `usage_daily_rollups` |
| provider 健康状态 | N/A | N/A | SQLite `provider_health` |
| provider 列表 | N/A | N/A | SQLite `providers` |
| session 状态 | hooks | hooks | hooks |

## 切换层级（运行时）

**用户可能改 settings.json 切层级：**

- 用户在 cc switch GUI 里切了个新 provider → cc switch 改写 settings.json env 块 → App 监听到 FSEvent → 重跑 SourceResolver → 仍是 L3（cc switch 还装着）
- 用户卸载 cc switch → App 监听不到 cc switch → 重跑 SourceResolver → 降级到 L1/L2
- 用户手动改 settings.json 的 env 块 → 同上

**App 怎么响应：**

```swift
// FSEvents on settings.json
func onSettingsJsonChanged() {
    let newSource = (try? sourceResolver.resolve()) ?? .official
    if newSource != appState.activeSource {
        appState.activeSource = newSource
        // 触发 HUD 完全 reload（项的可用性可能变）
        touchBar?.reloadAll()
        // 重启 / 停止相应的 poller
        poller.adjust(for: newSource)
    }
}
```

## Poller 按层级调度

```swift
final class Poller {
    func adjust(for source: ActiveSource) {
        switch source {
        case .official, .envVars:
            // 不查 SQLite，只靠 hooks + settings.json
            stopDBPolling()
            startSettingsWatch()  // 等 env 块变化
        case .ccSwitch:
            // 启动 SQLite 轮询
            startDBPolling(interval: 5)
            startSettingsWatch()  // cc switch 也可能改 env
        }
    }
}
```

## Edge cases

| 情况 | 处理 |
|---|---|
| 装了 cc switch 但 activeProvider 为 null | 降级到 L1/L2，HUD 显示 "cc switch 未配置 provider" |
| cc switch 正在切 provider（settings.json 半写状态） | SourceResolver 检测到 cc switch 存在就用 L3，避免读半截 env |
| env 块指向第三方但实际是 Claude 官方某个 alias | 不可识别，按 L2 处理（用户责任） |
| 同时有官方订阅 + cc switch（cc switch 配了官方 provider） | 走 L3（cc switch 数据更全），provider 名显示 cc switch 里的 |
| 用户改了 settings.json 后 App 没及时重算 | settings.json FSEvents 兜底；UI 加"手动重检测"按钮 |

## provider 友好名映射（L2 用）

```swift
let providerNameByHost: [String: String] = [
    "open.bigmodel.cn":       "Zhipu GLM",
    "api.deepseek.com":       "DeepSeek",
    "api.moonshot.cn":        "Moonshot Kimi",
    "dashscope.aliyuncs.com": "阿里通义",
    "api.mistral.ai":         "Mistral",
    "api.openai.com":         "OpenAI",
    // 兜底：用 hostname
]

func friendlyProviderName(from baseURL: String) -> String {
    guard let host = URL(string: baseURL)?.host else { return baseURL }
    return providerNameByHost[host] ?? host
}
```

用户可以在偏好设置里覆盖这个映射。
