## 数据源解析层（Source Resolver）

App 启动时按优先级探测数据源：

```
1. 读 ~/.claude/settings.json 的 env 块
   - 有 ANTHROPIC_BASE_URL ≠ 官方域名 → L2 第三方模型
   - env 为空或指向官方 → L1 官方订阅

2. 检测 cc switch（如果装了）
   - 装了且 activeProvider 存在 → L3 数据源激活，覆盖 L1/L2 的显示
   - 没装 → 不影响 L1/L2

3. 结果记入 AppState.activeSource（.official / .envVars / .ccSwitch）
   - HUD 据此决定显示哪些字段（cost 仅 L3 有）
```

详见 [03-数据源策略.md](03-数据源策略.md)。

## 数据流

### 输入路径

| 来源 | 通道 | 频率 | 内容 |
|---|---|---|---|
| `~/.claude/settings.json` | 文件 watch | 变化触发 | env 块、模型映射、hook 注册 |
| Claude.app / Claude Code 状态 | 启动检测 + hook 推断 | 启动 + 事件 | 订阅状态、session 信息（L1） |
| `cc-switch.db` | SQLite 只读 + FSEvents | 5s 轮询（仅 L3） | provider、usage、health、proxy |
| Claude Code hooks | JSONL 文件追加 + FSEvents | 事件驱动 | session 生命周期、状态变化 |
| NSWorkspace | `didActivateApplicationNotification` | 用户切窗口时 | frontmost app + 窗口 |

### 输出路径

```
AppState
  ├─▶ TouchBarController.touchBar.reload()
  └─▶ MainWindowController.tableView.reloadData()
```

### 副作用路径

| 动作 | 触发条件 |
|---|---|
| 启动 cc switch GUI | 用户点 Touch Bar / 主窗口的 `openCCS` 按钮（仅 L3） |
| 激活终端/IDE 窗口 | 用户在会话列表点击 `Activate` |
| 打开 App 主窗口 | 用户点 Touch Bar 的 `openApp` 按钮 |

## 核心数据模型

```swift
@Observable
final class AppState {
    // setup
    var setupComplete: Bool
    var lastError: String?

    // claude
    var claudeBinaryPath: String?
    var claudeConfigDir: URL?

    // 数据源
    var activeSource: ActiveSource           // .official / .envVars / .ccSwitch

    // cc switch（仅 activeSource == .ccSwitch 时有效）
    var ccSwitchAppURL: URL?
    var ccSwitchDBURL: URL?
    var ccSwitchAvailable: Bool

    // provider / model（按 activeSource 解析）
    var provider: ProviderInfo?              // 当前激活
    var modelMapping: ModelMapping?
    var availableProviders: [ProviderInfo]   // 仅 cc switch 提供

    // usage（按 activeSource 解析）
    var costToday: Double?                   // 仅 cc switch 有精确值
    var costMonth: Double?
    var requestCountToday: Int?

    // 多会话（来自 hooks）
    var sessions: [SessionID: Session]
    var focusedSessionID: SessionID?

    var lastRefreshedAt: Date?
}

enum ActiveSource: Equatable {
    case official                          // L1：官方订阅
    case envVars(baseURL: String)          // L2：第三方环境变量
    case ccSwitch                          // L3：cc switch
}

struct Session: Identifiable {
    let id: SessionID
    var cwd: URL
    var hostApp: HostApp
    var termSessionID: String?
    var tmuxPane: String?
    var status: SessionStatus
    var lastActivityAt: Date
    var startedAt: Date
    var endedAt: Date?
}

struct ProviderInfo: Equatable {
    let id: String
    let name: String
    let source: ActiveSource               // 来自哪一层
    let providerType: String?
    let limitDailyUSD: Double?
    let limitMonthlyUSD: Double?
}

struct ModelMapping: Equatable {
    let baseURL: String
    let haikuModel: String?
    let sonnetModel: String?
    let opusModel: String?
    let defaultModel: String?
}

enum SessionStatus: String { case idle, thinking, streaming, error, unknown }
enum HostApp: String { case iterm, terminalApp, vscode, cursor, ghostty, wezterm, tmux, unknown }
```

## 关键不变量

1. **Touch Bar 永远不访问文件系统或网络。** 所有数据通过 `AppState` 注入。
2. **Bridge 层的所有 I/O 都是可失败的。** 失败时写 `AppState.lastError`，不抛到 UI。
3. **`AppState.sessions` 的所有更新都在主线程。** Hook ingester 通过 `DispatchQueue.main.async` 派发。
4. **`activeSource` 是稳定派生态。** Setup 时计算一次，仅在 settings.json 变化时重算。
5. **cc switch 是可选的。** `activeSource == .ccSwitch` 时才查询 SQLite。

## 仅支持 Touch Bar 机型

App 启动时检测机型：

```swift
func isTouchBarAvailable() -> Bool {
    // 1. 检测 Touch Bar 支持的机型
    let model = MacModelIdentifier.current()  // e.g., "MacBookPro15,1"
    return TouchBarModels.supported.contains(model)
}

// 若不支持 → 弹窗提示并退出
```

不支持的机型：弹一次性提示 "cc-touchbar 仅支持带 Touch Bar 的 MacBook"，用户确认后退出。不做 fallback UI（菜单栏模式 / Dock 模式都不做）。
