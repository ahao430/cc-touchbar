# 10 - 风险与待办

## 风险登记表

| 风险 | 影响 | 缓解 |
|---|---|---|
| 用户机器没有 Touch Bar | App 无法工作 | 启动时机型检测，不支持就弹窗 + 退出 |
| `cc switch` 没装 | L3 不可用，但 App 仍能用（L1/L2） | Setup 标记为可选；HUD 隐藏 cost / openCCS 按钮 |
| cc switch schema 改动 | DTO 解析失败 | DTO 尽力匹配 + 每次启动校验 `sqlite_master`；decode 失败走 `lastError`，HUD 显示 |
| cc switch 配置目录被 override | 找不到 db | **必须**读 `app_paths.json`，不能假定 `~/.cc-switch/` |
| binary 路径随 nvm 升级失效 | "claude not found" 假阳性 | 持久化"nvm-managed"标记，Poller 周期重跑 `which claude` |
| cc switch 运行中改 db 不生效 | 直写 `is_current` 不被 reload | v1/v2/v3 严格区分；v3 才考虑直写，且需 reverse-engineer reload |
| SQLite WAL 锁 | 阻塞 cc switch 主进程 | 只用 `SQLITE_OPEN_READONLY` + WAL 模式，绝不写 |
| Hook 脚本未安装 | 看不到 session 实时状态 | Setup Wizard 一键安装；UI 提示 |
| Hook 进程被 CC 杀 / 卡住 | 事件丢失 | 单次 hook 失败不阻塞；TTL 兜底 |
| 用户拒绝合并 settings.json 的 hooks 块 | hook 接入失败 | Setup Wizard 用 jq 合并（不覆盖用户已有 hooks）；用户拒绝则降级到"无 session 状态"模式 |
| AppleScript 权限被拒 | 窗口激活失效 | 首次激活时弹权限对话框；拒绝后走 PID fallback |
| 多个 CC 同时写 jsonl | 行交错（理论上） | POSIX append 单次 write < 4KB 原子；超过的话用临时文件 + cat |
| jsonl 文件无限增长 | 磁盘占用 | App 启动时清理 7 天前的行；CC 进程不依赖旧数据 |
| 用户改了 cc switch override 但 App 还在用旧路径 | 数据陈旧 | 每次启动都重新解析；FSEvents 监听 `app_paths.json` 变化 |
| VS Code / Cursor 等不暴露终端 tab ID | 窗口激活不准 | fallback：URL scheme + 窗口标题模糊匹配 basename(cwd) |
| Claude Code 升级后 hook 字段变化 | jsonl 解析失败 | DTO 尽力匹配；未知字段忽略；保留原始 raw 字段用于调试 |
| CC 在 SSH 远端跑 | 无法激活远端窗口 | hostApp 检测为 `remote`，Activate 按钮禁用，显示 host |
| 长任务后台 streaming，用户在别处 | HUD 跟随后台 session（可能不期望） | 这是设计行为；用户可以点 openApp 切回主窗口 |
| App 主窗口和 Touch Bar 状态分裂 | 用户困惑 | 共享同一 AppState；操作任一处都同步 |
| Claude Code 官方改订阅模式（按 token 计费） | L1 的"无 cost 数据"假设失效 | SourceResolver 增加 cost 检测；hook 累计 token |
| settings.json 半写状态（cc switch 切 provider 中） | SourceResolver 误判 | 优先检测 cc switch 存在 → L3（不读半截 env） |

## 待办问题（开工前需要确认）

### 高优先级（阻塞 v1）

1. **Touch Bar 机型完整清单**
   - MacBook Pro 2016-2019 哪些型号有 Touch Bar？
   - MacBook Air（任何年份）都无 Touch Bar
   - Apple Silicon 之后（2021+）的 MacBook Pro 都无 Touch Bar
   - 外接 Magic Keyboard with Touch Bar 怎么检测？
   - 影响：v1 的机型白名单

2. **Claude Code 官方订阅怎么从本地拿状态？**
   - `~/.claude/` 下有没有订阅状态文件？
   - 是否能从 `claude` CLI 调用某 API 拿到 Pro/Max 状态？
   - 影响：L1 数据源的 provider 信息丰富度

3. **statusline-command.sh 是不是用户机器特有？**
   - 当前用户机器有，但其它用户未必
   - 如果是统一的，要在 Setup Wizard 里安装
   - 影响：是否复用用户已有 statusline 输出

### 中优先级（影响 v2）

4. **AppleScript 在 sandbox 环境下的权限模型**
   - Mac App Store 上架会严格限制
   - Developer ID 直发可能也需要用户授权"自动化"
   - 影响：分发方式决策

5. **cc switch 持续 schema 演进**
   - 当前表结构是否会变？
   - 是否有版本字段？
   - 影响：DTO 兼容性策略

6. **`MessageDisplay` hook 的实际触发频率**
   - 文档没明说"per batch"具体多少
   - 太频繁 → jsonl 写入压力大
   - 影响：是否要在 hook 脚本里 throttle

7. **Claude Code 的 hook 配置怎么和用户已有 hooks 合并？**
   - 直接覆盖会破坏用户其它工具
   - jq 合并 + 标记（如 `"// cc-touchbar-start"`）？
   - 影响：Setup Wizard 实现

### 低优先级（影响 v3）

8. **cc switch 启动 / reload 信号是什么？**
   - SIGHUP？特定端口调用？需要 reverse-engineer Tauri 进程
   - 影响：v3 的"直写 SQLite 切 provider"能否落地

9. **cc switch 是否暴露 deeplink / 本地 HTTP API？**
   - `lsof -i :15721` 只看到代理端口
   - Tauri 应用可能有自定义 URL scheme
   - 影响：v3 的 `openUI(toProvider:)` 能否精准跳转

10. **codex / gemini 的 hook 体系**
    - cc switch 支持，但它们自己有类似 hook 吗？
    - 影响：多 app_type 的实时状态接入

11. **使用分析的数据源**
    - `usage_daily_rollups` 够做日级图表
    - 按小时？按模型？要查 `proxy_request_logs`（数据量大）
    - 影响：图表设计

12. **DFR（Dynamic Function Row）私有 API**
    - Apple 没公开文档但社区有用法
    - App Store 上架会被拒（私有 API）
    - 影响：v3 的"始终显示 HUD"功能可行性

## 已知陷阱速查

| 陷阱 | 说明 |
|---|---|
| 不要假定 `~/.ccswitch/` 存在 | 老版本路径；当前默认 `~/.cc-switch/`；用户可 override |
| 不要假定 Claude binary 在 `/usr/local/bin` | nvm 管理时只在 nvm shim 里 |
| 不要假定 iTerm 是用户唯一终端 | Ghostty / WezTerm / VS Code / Terminal.app 都要支持 |
| 不要假定 cc switch 是 CLI | 是 Tauri GUI，无子命令 |
| 不要假定 cc switch 装了 | L1/L2 用户都没装 |
| 不要假定 `MessageDisplay` = token 流 | 文档说 "per-batch"，需要实测 |
| 不要假定 hook 进程有 TTY | v2.1.139+ hook 在独立 session 跑 |
| 不要假定 settings.json 的 env 块不会变 | cc switch 改 provider 时会改写 |
| 不要缓存 db 路径超过一次会话 | override 可能随时改 |
| 不要假定所有 MacBook 都有 Touch Bar | 2021 年后基本都没了 |

## 监控与诊断

App 应该提供：

- **诊断面板**（偏好设置里）：
  - 持久化的所有路径
  - 当前 `activeSource` + 推断依据
  - 当前 db schema 校验结果（仅 L3）
  - hook 脚本是否已注册（解析 settings.json）
  - 最近 10 条 hook 事件（debug 用）
  - 最近一次错误 + 完整 stack trace
  - "复制诊断信息"按钮（用户报告 bug 时贴给开发者）

- **log 文件**：
  - `~/Library/Logs/cc-touchbar/app.log`
  - 按天 rotate，保留 7 天
  - 默认 info 级，可调 debug
