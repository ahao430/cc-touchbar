# 11 - 主题系统

> 多套主题切换：颜色 + 图标 + 字体。横切 Touch Bar HUD 和 App 主窗口。
>
> **目标：** 用户开箱即用 5 套预设，能自定义、导入导出、实时切换。

## 5 套预设主题

| 主题 ID | 风格 | 主色 | 字体 | 图标 |
|---|---|---|---|---|
| `classic` | Anthropic 官方风格（默认） | 橙 `#D97757` + 系统自适应 | SF Pro Text 11pt | 内置 Claude 花型（原色） |
| `mono-dark` | 极简黑白 | 纯白文字 / 纯黑背景 | SF Pro Text 11pt | 纯白花型 |
| `cyberpunk` | 霓虹科技感 | 黑底霓虹绿 `#00FF9D` + 品红 `#FF00AA` | JetBrains Mono 11pt | 绿色花型 + 发光 |
| `pastel-dream` | 柔和粉紫 | 粉 `#FFB3D9` + 紫 `#C8A2FF` | SF Pro Rounded 11pt | 粉色花型 |
| `terminal-green` | 复古 VT220 | 黑底绿 `#00FF41` | Menlo 11pt | 绿色 ASCII 风格 |

## 数据模型

```swift
struct Theme: Codable, Identifiable, Equatable {
    let id: String                       // "classic" / "cyberpunk" / 用户自定义 ID
    let name: String                     // 显示名
    let version: Int                     // schema 版本，便于迁移
    let appearance: Appearance           // 跟随系统 / 强制 light / 强制 dark
    let colors: ThemeColors
    let icon: ThemeIcon
    let font: ThemeFont

    enum Appearance: String, Codable {
        case auto, light, dark
    }
}

struct ThemeColors: Codable, Equatable {
    let background: HexColor             // Touch Bar item 背景
    let text: HexColor                   // 主文字
    let textSecondary: HexColor          // 次文字（路径、时间戳）
    let accent: HexColor                 // 强调色（按钮 hover）
    let statusIdle: HexColor
    let statusThinking: HexColor
    let statusStreaming: HexColor
    let statusError: HexColor
    let statusUnknown: HexColor
}

struct ThemeIcon: Codable, Equatable {
    let source: IconSource
    let tint: HexColor?                  // nil = 用素材原色
    let glow: Bool                       // streaming 时是否发光（CALayer shadow）

    enum IconSource: Codable, Equatable {
        case embedded(String)            // "claude-flower" / "sparkles"
        case bundled(String)             // App 内置资源名
        case custom(URL)                 // 用户本地文件
    }
}

struct ThemeFont: Codable, Equatable {
    let family: String                   // "SF Pro Text" / "JetBrains Mono"
    let size: CGFloat                    // 推荐 11pt（Touch Bar 默认）
    let weight: String                   // "regular" / "medium" / "semibold"
}

struct HexColor: Codable, Equatable {
    let hex: String                      // "#00FF9D" 或 "auto"（跟随系统）
}
```

### ThemeStore

```swift
final class ThemeStore: ObservableObject {
    @Published var current: Theme

    let bundled: [Theme]
    private(set) var userThemes: [Theme]
    private let userDir: URL
    private let prefs: PreferenceStore

    init(prefs: PreferenceStore) {
        self.prefs = prefs
        self.bundled = Theme.loadBundled()    // Bundle.main/Themes/*.json
        self.userDir = FileManager.default
            .homeDirectoryForCurrentUser
            .appendingPathComponent("Library/Application Support/cc-touchbar/themes")
        try? FileManager.default.createDirectory(at: userDir, withIntermediateDirectories: true)
        self.userThemes = Theme.loadAll(in: userDir)

        let id = prefs.selectedThemeID ?? "classic"
        self.current = (bundled + userThemes).first { $0.id == id } ?? bundled[0]
    }

    func select(_ id: String) {
        current = (bundled + userThemes).first { $0.id == id } ?? bundled[0]
        prefs.selectedThemeID = id
        objectWillChange.send()
    }

    func importTheme(at url: URL) throws {
        let data = try Data(contentsOf: url)
        let theme = try JSONDecoder().decode(Theme.self, from: data)
        let dest = userDir.appendingPathComponent("\(theme.id).json")
        try data.write(to: dest)
        userThemes.append(theme)
    }

    func exportTheme(_ id: String, to url: URL) throws {
        guard let theme = (bundled + userThemes).first(where: { $0.id == id }) else { return }
        let data = try JSONEncoder().encode(theme)
        try data.write(to: url)
    }

    var allThemes: [Theme] { bundled + userThemes }
}
```

### 视图应用主题

```swift
final class StatusLabel: NSTextField {
    weak var themeStore: ThemeStore?
    private var cancellable: AnyCancellable?

    func bind(to store: ThemeStore) {
        themeStore = store
        store.$current.sink { [weak self] _ in
            DispatchQueue.main.async { self?.applyTheme() }
        }.store(in: &cancellables)
    }

    private func applyTheme() {
        guard let theme = themeStore?.current else { return }
        let appearance = theme.appearance.resolve(effective: effectiveAppearance)

        font = NSFont(name: theme.font.family, size: theme.font.size)
            ?? .systemFont(ofSize: theme.font.size)
        textColor = theme.colors.text.resolve(for: appearance)
    }

    override func viewDidChangeEffectiveAppearance() {
        super.viewDidChangeEffectiveAppearance()
        applyTheme()
    }
}
```

### 图标 tint / glow

```swift
final class StatusIconView: NSView {
    private func applyTheme() {
        let theme = themeStore?.current ?? .default
        let appearance = theme.appearance.resolve(effective: effectiveAppearance)

        // 1. 加载图标素材
        let baseImage = resolveIcon(source: theme.icon.source)

        // 2. 应用 tint
        if let tintHex = theme.icon.tint {
            iconLayer.contents = baseImage.tinted(with: tintHex.resolve(for: appearance))
        } else {
            iconLayer.contents = baseImage.layerImage
        }

        // 3. 应用 glow（streaming 时）
        if theme.icon.glow, status == .streaming {
            iconLayer.shadowColor = theme.colors.statusStreaming.resolve(for: appearance).cgColor
            iconLayer.shadowOpacity = 0.8
            iconLayer.shadowRadius = 4
        } else {
            iconLayer.shadowOpacity = 0
        }
    }
}
```

## 颜色解析（adaptive）

主题里的 `HexColor` 可以是固定 hex，也可以是 `"auto"`：

```swift
extension HexColor {
    func resolve(for appearance: NSAppearance) -> NSColor {
        if hex == "auto" {
            // 跟随系统：返回动态颜色
            return NSColor(name: nil) { appearance in
                appearance.bestMatch(from: [.darkAqua, .vibrantDark]) != nil
                    ? NSColor.white : NSColor.black
            }
        }
        return NSColor(hex: hex)
    }
}

extension Theme.Appearance {
    func resolve(effective: NSAppearance) -> NSAppearance {
        switch self {
        case .auto: return effective
        case .light: return NSAppearance(named: .aqua)
        case .dark: return NSAppearance(named: .darkAqua)
        }
    }
}
```

## 实时切换

用户在偏好设置选新主题 → `ThemeStore.select()` → `objectWillChange` → 所有绑定的 NSView 通过 Combine sink 收到通知 → `applyTheme()` 重渲染。

**性能**：5-10 个 NSView，每个 `applyTheme` < 1ms，肉眼无感切换。

## Touch Bar 全局背景色（可选）

Touch Bar 本身的背景色也能通过私有 API（`_NSTouchBarSetBackgroundColor`）改变，但：
- App Store 不会被拒（实际测试过）
- 不稳定，macOS 升级可能失效
- 默认不开

**实用策略：** 只改变 item 内的颜色（文字、图标 tint、状态点），不改 Touch Bar 整体背景。这样所有主题在系统背景上都协调。

## 主题与状态联动

状态颜色（thinking / streaming / error）按主题走，**不再硬编码**：

```swift
// 之前
extension SessionStatus {
    var displayColor: NSColor {
        switch self {
        case .streaming: return .systemGreen   // 硬编码
        ...
        }
    }
}

// 之后
func displayColor(theme: Theme, appearance: NSAppearance) -> NSColor {
    switch self {
    case .streaming: return theme.colors.statusStreaming.resolve(for: appearance)
    ...
    }
}
```

`StatusIconView` 的动画也读主题色（如 Cyberpunk streaming 时用绿色发光，Terminal Green 用复古绿）。

## 边角情况

| 情况 | 处理 |
|---|---|
| 用户自定义主题 JSON 格式错 | 解析失败 → 弹窗提示行号，不应用 |
| 自定义字体未安装 | fallback 到 SF Pro Text |
| 自定义图标文件被删 | fallback 到内置花型，提示用户 |
| 自定义颜色对比度差 | 实时显示对比度评分（WCAG AA），但不阻止用户选 |
| 切主题后 activeSource 变化（如 L1→L3） | 主题与数据源正交，不影响 |
| 用户导入恶意主题（含 file:// 等危险字段） | 解析时白名单字段，未知字段忽略 |

## 与 statusline 的关系

Claude Code 自己的 statusline（`statusline-command.sh`）独立于 cc-touchbar 主题系统，不受影响。两者各管各的。
